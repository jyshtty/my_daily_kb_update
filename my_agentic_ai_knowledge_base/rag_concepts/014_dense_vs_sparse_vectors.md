# Dense vs Sparse Vectors

> From notebook: `similarity_search_essentials.ipynb`

## Dense Vectors
Generated by neural network embedding models. Most/all elements are non-zero.

- Capture **semantic meaning** — "car" and "automobile" are close in vector space
- Fixed dimension (e.g. 256, 512, 768, 1024) regardless of vocabulary
- Single dimension means nothing alone; meaning emerges from all dims together
- Great for: general semantic search, RAG baseline

```python
from langchain_huggingface import HuggingFaceEmbeddings

embedding_function = HuggingFaceEmbeddings(
    model_name="sentence-transformers/all-MiniLM-L6-v2",
    model_kwargs={"device": "cpu"},
    encode_kwargs={"normalize_embeddings": False},
)

embedded_chunks = embedding_function.embed_documents(chunks)
```

## Sparse Vectors
Each dimension corresponds to a word/subword token. Most elements are zero.

- Capture **exact term matching** — essential for rare/domain-specific terms
- Dimension = vocabulary size (tens of thousands)
- Fail at semantic nuance: "king" ≠ "queen" in sparse space
- Great for: medical/legal/technical domains, keyword-heavy queries

### BM25 (Classic Sparse)
Built into Elasticsearch, Pinecone, Milvus — no custom implementation needed.

### BGE-M3 (Neural Sparse + Dense)
Returns both sparse (lexical weights) and dense vectors from one model:
```python
from FlagEmbedding import BGEM3FlagModel

model = BGEM3FlagModel("BAAI/bge-m3", use_fp16=True)
output = model.encode(sentences, return_dense=False, return_sparse=True)

sparse_weights = output["lexical_weights"]
tokens = model.convert_id_to_token(sparse_weights)
```

### SPLADE (Learned Sparse with Term Expansion)
Expands query/document with contextually relevant terms not in the original text.
Example: "solar energy advantages" → also indexes "renewable", "sustainable", "photovoltaic"
```python
from transformers import AutoModelForMaskedLM, AutoTokenizer

model_id = "naver/splade-cocondenser-ensembledistil"
tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForMaskedLM.from_pretrained(model_id)

def compute_vector(text, tokenizer, model):
    tokens = tokenizer(text, return_tensors="pt")
    output = model(**tokens)
    logits, attention_mask = output.logits, tokens.attention_mask
    relu_log = torch.log(1 + torch.relu(logits))
    weighted_log = relu_log * attention_mask.unsqueeze(-1)
    max_val, _ = torch.max(weighted_log, dim=1)
    return max_val.squeeze(), tokens
```

## Comparison Table

| Feature | Sparse | Dense |
|---|---|---|
| Most elements | Zero | Non-zero |
| Captures | Exact terms | Semantic meaning |
| Rare/domain terms | Excellent | Poor |
| Semantic similarity | Poor | Excellent |
| Examples | BM25, SPLADE, BGE-M3 sparse | OpenAI, SBERT, BGE-M3 dense |
| Use in RAG | Hybrid search component | Primary retrieval |

---

## Hybrid Search
Combine sparse + dense for best of both worlds:
- Dense retrieves semantically similar chunks
- Sparse ensures exact keyword matches aren't missed
- Common in Qdrant, Pinecone, Elasticsearch, Milvus

## Visualizing Embeddings with UMAP
Reduce high-dimensional embeddings to 2D/3D for visualization:
```python
import umap.umap_ as umap
import numpy as np

embedded_chunks_np = np.array(embedded_chunks)
umap_model = umap.UMAP(n_components=2, random_state=42)
umap_embeddings = umap_model.fit_transform(embedded_chunks_np)
```
Useful for verifying that embeddings cluster by topic — semantically similar chunks should group together.

SPLADE expands terms — 60 tokens indexed for a short chunk, including terms not in the original text. This **term expansion** is what makes it more powerful than plain BM25.
